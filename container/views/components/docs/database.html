Database

Chasi can handle multiple MongoDB connection, the connections will be initialized on boot.
the instance also implements Mongoose, a schema-based solution to model.
be sure to check their website:
https://mongoosejs.com/

Note: As of this moment, Chasi only supports MongoDB as it’s database, Although developers can always implement their own database service

Check the DB config in “./config/database.js”, where you can setup the DB environment,
and the connections

connection setup may look like this:

local:  {
    url:  checkout(process.env.dbConStringLocal),
    db:  checkout(process.env.databaseName),
    options:  {
      useNewUrlParser:  true,
      useUnifiedTopology:  true,
      useCreateIndex:  true
    }
  }
  • url [string] - the database host address
  • db [string] - the collection name
  • options [object] - extensive settings for the database instance and connection
    check: https://mongoosejs.com/docs/connections.html

Choosing connections for a model

you can do this by adding the connection as a parameter in “.getConnection()” method, inside a model.

default connection will be used when the declared connection is not existing.

E.G.:

const User = Adapter.getConnection("local").model('users', UserSchema);
  module.exports  = User;
